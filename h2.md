## a) a) Apache log

Asenna Apache-weppipalvelin paikalliselle virtuaalikoneellesi. Surffaa palvelimellesi salaamattomalla HTTP-yhteydellä, http://localhost . Etsi omaa sivulataustasi vastaava lokirivi. Analysoi yksi tällainen lokirivi, eli selitä sen kaikki kohdat. (Jos Apache ei ole kovin tuttu, voit tätä tehtävää varten vain asentaa sen ja testata oletusweppisivulla. Eli ei tarvitse tehdä omia kotisvuja tms.)

Asennetaan paikalliselle virtuaalikoneelle Apache -webpalvelin ja tarkastellaan sen lokirivejä.

Aloitetaan asentamalla apache2.

    sudo apt install apache2

Apache on webpalvelin, joka toimii taustalla. Sitä voi hallita systemctl -komennon avulla.

Käynnistetään palvelin.

    sudo systemctl start apache2

Varmistetaan vielä, että Apache käynnistyy aina koneen käynnistyessä.

    sudo systemctl enable apache2

Nyt voidaan mennä selaimessa localhost -osoitteeseen salaamattomalla yhteydellä:

    http://localhost

Apachen oletussivu ilmestyy näkyviin.

<img width="1000" height="548" alt="image" src="https://github.com/user-attachments/assets/20f5801e-7142-4b36-9001-7291e44d544c" />

Etsitään seuraavaksi lokirivi, jolla näkyy websivun lataus. Siirrytään Apachen lokihakemistoon.

    cd /var/log/apache2

<img width="864" height="121" alt="image" src="https://github.com/user-attachments/assets/50cfa611-d654-4261-9fde-13c13840dc1f" />

Tarkastellaan access.log -tiedostoa. tail -komennolla seurataan lokitiedoston perää ja -f -valinnalla voidaan seurata lokia reaaliajassa. Laitetaan pari enterin lyöntiä, ja käydään lataamassa localhost -sivu selaimessa. Tällöin nähdään mitkä lokitiedot ilmestyy kyseisestä HTTP -pyynnöstä.

    tail -f access.log

<img width="963" height="500" alt="image" src="https://github.com/user-attachments/assets/2a4645ac-5ec6-4797-a326-5b768ef8145d" />

Analysoidaan punaisen laatikon sisällä oleva HTTP-pyyntö.

- **127.0.0.1** on IP-osoite, josta pyyntö tuli. Tässä se on localhost-osoite eli sen websivun näkee ainoastaan omalla koneella paikallisesti eikä ole saavutettavissa internetin välityksellä.
- **[08/Apr/2026:21:47:42 +0300]** on aika, jolloin HTTP-pyyntö on tehty. Siinä näkyy päivämäärä, kellonaika ja UTC.
- **"GET / HTTP/1.1"** tarkoittaa, että käytetään HTTP -protokollaa ja tehdään GET -pyyntö palvelimelle.
- **200** on HTTP-statuskoodi ja tässä tapauksessa se tarkoittaa, että HTTP-pyyntö on onnistunut.
- **3383** on clientille lähetetyn vastauksen koko.
- **"Mozilla/5.0 (X11; Linux x86_64; rv:140.0) Gecko/20100101 Firefox/140.0"** kertoo HTTP-pyynnön tekijän eli clientin selaintiedot. Käyttäjä on käyttänyt selaimen Mozilla Firefoxia, koneen käyttöjärjestelmä on ollut Linux. ([David Girvin 2025: Understanding the Apache access log: how to view, locate, and analyze](https://www.sumologic.com/blog/apache-access-log))

## Nmapped

Porttiskannaa oma weppipalvelimesi käyttäen localhost-osoitetta ja 'nmap -A' päällä. Selitä tulokset. (Pelkkä http-portti 80/tcp riittää)

Porttiskannataan oma localhost-osoitteessa oleva webpalvelin Nmapin avulla ja tarkastellaan tuloksia. Nmap eli Network Mapper on työkalu, jonka avulla voidaan verkosta löytää laitteita, tunnistaa avoimia portteja, havaita palveluja ja kerätä tietoa verkon käyttöjärjestelmistä. Sitä voidaan käyttää tietoturvan arvioinnissa ja vianmäärityksessä ([GeeksForGeeks 2025: What Is Nmap? A Comprehensive Guide For Network Mapping](https://www.geeksforgeeks.org/linux-unix/what-is-nmap-a-comprehensive-guide-for-network-mapping/))

Irrotetaan virtuaalikone internetistä varmuuden vuoksi, ettei vahingossakaan porttiskannata kenenkään muun laitetta, koska se on laitonta.

Kun kone on irrotettu internetistä, varmistetaan se koittamalla pingata Googlen DNS-palvelinta, jonka IP-osoite on 8.8.8.8.

<img width="708" height="128" alt="image" src="https://github.com/user-attachments/assets/8bbc9ee7-3787-4a28-a074-90944b5c24ee" />

Vastausta ei tullut, joten voidaan todeta, että kone on irrotettu internetistä.

Suoritetaan localhost -osoitteen porttiskannaus.

        nmap -A localhost

-A valinnan avulla nmap ottaa käyttöön käyttöjärjestelmän ja sen version havaitsemisen, skriptien skannauksen ja tracerouten.

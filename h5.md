Mininet

## a) ARP poisoning

Tehtävänä on ajaa ARP-hyökkäys ja tutkia, kuinka se toimii.

Päivitetään ensin pakettilistaus sekä päivitettävissä olevat paketit.

    sudo apt update
    sudo apt upgrade

Kokeillaan käynnistää verkko-ohjain.

    ryu-manager ryu.app.simple_switch_13

Ei onnistunut. Tulosteessa kehotettiin asentamaan python3-ryu paketti.

    sudo apt install python3-ryu

Kokeillaan uudestaan käynnistää verkko-ohjain.

    ryu-manager ryu.app.simple_switch_13

Avataan toinen terminaali, jossa käynnistetään mininet.

    sudo mn --topo single,3 --mac --switch ovsk --controller remote

<img width="956" height="634" alt="image" src="https://github.com/user-attachments/assets/c7794914-a4e1-449f-8240-ea97edd65573" />

Tämä mininet emuloi verkkoa, jossa on kolme päätelaitetta (h1, h2, h3) ja kytkin (s1). Päätelaitteet on yhdistetty kytkimeen.

Testataan, että kaikki päätelaitteet voivat kommunikoida keskenään.

    pingall

Tuloksena oli *0% dropped (6/6 received)* eli päätelaitteilla on toisiinsa toimivat yhteydet.

Tarkistetaan h1 ARP-taulukko. Sen tehtävänä on siis linkittää fyysiset eli MAC-osoitteet ja loogiset eli IP-osoitteet toisiinsa. 
Tämä tapahtuu ARP-kyselyiden kautta. Kone siis lähettää broadcast-viestin, jossa se pyytää tietyn IP-osoitteen omaavaa laitetta kertomaan MAC-osoitteensa.
Kun se saa tämän tiedon, se lisää sen ARP-taulukkoonsa. ARP-taulukon avulla koneen ei tarvitse aina lähettää ARP-kyselyä vaan voi lukea
tiedon taulusta. ARP-taulukko ei ole pysyvä vaan sieltä poistuu rivejä tietyn ajan kuluessa.

<img width="1596" height="162" alt="image" src="https://github.com/user-attachments/assets/9310b41a-0b53-429e-b840-63af1c28af1b" />

ARP-taulukosta voidaan päätellä, että siellä on h2 ja h3 koneiden IP- ja MAC-osoitteet.

Kytketään päälle IP Forwarding eli IP-pakettien välitys h3 koneella. Tämä mahdollistaa sen, että h3 kone voi välittää verkossa olevia paketteja.

    sysctl -w net.ipv4.ip_forward=1

- sysctl komentoa käytetään, kun halutaan muokata kernelin parametrejä ajon aikana ([Linux manual page: sysctl(8)](https://man7.org/linux/man-pages/man8/sysctl.8.html)).
- -w parametri kirjoittaa argumentille uuden arvon ([Linux manual page: sysctl(8)](https://man7.org/linux/man-pages/man8/sysctl.8.html)).
- net.ipv4.ip_forward=1 kytkee IP-pakettien välityksen päälle, koska sen arvoksi annetaan 1 ([OpenVPN: Check Linux IP Forwarding for Access Server Routing](https://openvpn.net/as-docs/faq-ip-forwarding-on-linux.html#check-linux-ip-forwarding-for-access-server-routing)).


Force all arguments to be write arguments and print an error if they cannot be parsed this way.

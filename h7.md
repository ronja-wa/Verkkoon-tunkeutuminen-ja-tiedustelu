# [h7 Ohjelmistoradio](https://terokarvinen.com/verkkoon-tunkeutuminen-ja-tiedustelu/)

## x) Lue ja tiivistä

### [Hubacek 2019: Universal Radio Hacker SDR Tutorial on 433 MHz radio plugs](https://www.youtube.com/watch?v=sbqMqb6FVMY&t=199s)

- URH eli Universal Radio Hacker on ohjelmistoradio, jonka avulla nauhoittaa radiosignaaleja.
- Radiosignaalit ilmaisevat bittejä ja yhdellä bitillä on tietty pituus millisekunteina. URH osaa automaattisesti määrittää bitin pituuden.
- URH voi muuttaa radiosignaalit biteiksi ja heksadesimaaleiksi.

### [Cornelius 2022: Decode 433.92 MHz weather station data](https://www.onetransistor.eu/2022/01/decode-433mhz-ask-signal.html)

- Rakennetaan sääasemalle sensorin hyödyntäen tietoa, että sääasema toimii taajuudella 433,92 MHz.
- Ensin kuunneltiin rtl-433 ohjelmalla sensoreiden lähettämää signaalia ja etsittiin lisää tietoa laitteen mallin perusteella.
- URHlla tallennettiin sääaseman sensorin signaalit ja analysoitiin sitä.
- Analyysissa määriteltiin bitit 0 ja 1, tarkasteltiin pulssien pituuksia ja lopuksi yhdistettiin URHn ja rtl-433n antama data, jolloin saadaan tietää signaalin rakenne.

## a) Lähteet ja läppä

Lähteet tarkistettu ja virtuaalikone valmis lipunryöstöön.

## b) rtl_433

rtl_433 on ohjelma, joka analysoi automaattisesti radiosignaaleja. Se pystyy tulostamaan sellaisten signaalien sisällön, joiden protokollan se tuntee. Ohjelmalla voi löytää esim. rengaspaineantureita, lämpömittareita tai smart home laitteita.

Asennetaan rtl_433 virtuaalikoneelle.

    sudo apt update
    sudo apt install rtl-433

<img width="1334" height="187" alt="image" src="https://github.com/user-attachments/assets/61f5811b-b63e-46fe-9cf8-46bdac263446" />

Koneeseen ei ole liitetty laitetta, jolla voisi signaaleja vastaanottaa, joten komento tulostaa ainoastaan ohjelman version.

## c) Automaattinen analyysi

Tehdään rtl_433 ohjelmalla automaattinen analyysi [Tero Karvisen sivulta](https://terokarvinen.com/verkkoon-tunkeutuminen-ja-tiedustelu/#tehtavanannot) löytyvään tiedostoon **Converted_433.92M_2000k.cs8**. Ladataan se ensin virtuaalikoneelle.

Tiedosto löytyy hakemistosta /home/kantarelli/Downloads.

        cd Downloads

Avataan tiedosto Converted_433.92M_2000k.cs8 rtl_433 ohjelmalla.

        rtl_433 -r Converted_433.92M_2000k.cs8

-r parametrilla valitaan tiedosto, josta saadaan data rtl_433 ohjelmaan (Linux manual: RTL_433(1)).

<img width="711" height="300" alt="image" src="https://github.com/user-attachments/assets/68e021f1-e2cc-45c3-ade7-649cc20e0594" />

Ohjelma listaa tiedostosta kaikki erilliset signaalit, joita se on havainnut. Signaalit ovat käyttäneet jotain rtl_433 ohjelman tuntemaa protokollaa, jolloin ohjelma on automaattisesti pystynyt kääntämään signaalin luettavaan muotoon.

Ohjelma löysi kolmea eri laitemallia:

- KlikAanKlikUit-Switch
- Proove-Security
- Nexa-Security

Netistä hakemalla löytyi, että nämä laitteet voisivat olla smart home laitteita.

Tiedostosta löytyy yhteensä 12 riviä eri signaaleja, mutta vain kolmea erilaista. Jokaista eri mallin signaalia tulee aina kerran, jonka jälkeen ne tulevat uudestaan. Tiedostossa näin tapahtuu yhteensä neljä kertaa.

time -kohdassa näkyy myös, että kaikki kolme signaalia tapahtuu täsmälleen samaan aikaan. Esim. ensimmäisessä kolmessa rivissä time arvo on 0.083284s, seuraavassa kolmessa 0.163125s

KlikAanKlikUit-Switch mallissa on eri tunnisteet kuin Proovessa ja Nexassa. House Code ja id taitavat kuitenkin tarkoittaa näissä kaikissa samaa, koska kaikissa niissä on sama arvo eli 8785315.

KlikAanKlikUit-Switchissä on Command -kohtaan annettu arvoksi Off, ja Proovessa ja Nexassa on State -kohdassa myös Off. Nämä kohdat voisivat ehkä vastata toisiaan.

Sivulta [trig.org: DECODER & SAMPLE EXPLORER](https://triq.org/explorer/) löytyy eri radiosignaalien protokollia dekoodattuna. Sieltä löytyi ainakin, että KlikAanKlikUit, Proove-Security ja Nexa-Security toimivat samanlaisella protokollalla, koska ne voidaan dekoodata samalla tavalla.

<img width="383" height="300" alt="image" src="https://github.com/user-attachments/assets/8ac327fe-b3a5-4ff2-a6ad-93fef6bc4816" />

Kaiken edellä mainitun perusteella voisin siis päätellä, että kyseessä voisi ollakin vain yksi laite, joka lähettää neljä signaalia. rtl_433 tulkitsee sen kuitenkin kolmen eri protokollan kautta.

## d) Too compex 16?

Muutetaan URH -ohjelmalla nauhoitettu .complex16s -tiedosto toiseen tiedostomuotoon, jotta sitä voidaan analysoida rtl_433 ohjelmalla.

Ladataan virtuaalikoneelle [Tero Karvisen sivulta](https://terokarvinen.com/verkkoon-tunkeutuminen-ja-tiedustelu/#tehtavanannot) löytyvä tiedosto **Recorded-HackRF-20250411_183354-433_92MHz-2MSps-2MHz.complex16s**.

Tiedosto löytyy hakemistosta /home/kantarelli/Downloads.

        cd Downloads

c-kohdan tehtävän tiedostosta **Converted_433.92M_2000k.cs8** voidaan tarkistaa, että rtl_433 ohjelmalla voidaan ainakin avata .cs8 -päätteinen tiedosto.

Tiedoston nimestä nähdään myös, että signaalin taajuus on 433,92 MHz ja 2000k tarkoittaa, kuinka paljon näytteitä lähetetään ([merbanan: Introduction to I/Q formats](https://github.com/merbanan/rtl_433/blob/ac1e4a8c5a36fb90e3b06c0f01cef00bb3b2614d/docs/IQ_FORMATS.md#file-name-meta-data))

Teron vinkeissä tehtävänannossa lukee, että signaalin taajuuden sekä näytteenottotaajuuden tulee olla oikea taajuus. Käännettävässä tiedostossa ja uudessa tiedosssa tulee siis olla sama taajuus.

Käännettävän tiedoston taajuus on 433,92 MHz ja näytteenottotaajuus on 2MSps eli 2000k näytettä sekunnissa.

Käännetään tiedosto .complex16s &rarr; .cs8

        cp Recorded-HackRF-20250411_183354-433_92MHz-2MSps-2MHz.complex16s convertedHackRF_433.92M_2000k.cs8

Tarkistetaan ls -komennolla, että uusi tiedosto ilmestyi.

<img width="424" height="80" alt="image" src="https://github.com/user-attachments/assets/c4275105-1973-4a2e-814d-414924f429dc" />

Kokeillaan avata se rtl_433 ohjelmalla.

        rtl_433 -r convertedHackRF_433.92M_2000k.cs8

Tiedosto aukesi.

<img width="768" height="400" alt="image" src="https://github.com/user-attachments/assets/574ac7c0-4fd3-41e3-806d-1c3d6cfacda6" />

Tiedosto on sama kuin c-kohdassa, koska niiden tiivisteet ovat täsmälleen samat. Analysointi on siis tehty jo c-kohdassa.

## e) Ultimate

Asennetaan URH eli Universal Radio Hacker. Asennetaan ensin pipx, jolla voidaan asentaa URH.

        sudo apt update
        sudo apt -y install pipx

Asennetaan URH.

        pipx install urh

Pitää vielä varmistaa, että kaikki pipx:n tarvitsemat hakemistot ovat PATH ympäristömuuttujassa. Tällöin pipx:llä asennetut ohjelmat toimivat terminaalissa kuten muutkin (Generated Python Manual: PIPX(1)).

        pipx ensurepath

Suljetaan terminaali ja avataan se uudestaan.

Avataan URH.

        urh

<img width="724" height="110" alt="image" src="https://github.com/user-attachments/assets/2bbe902d-9882-43d2-88e3-f1a5dd93c93e" />

Tulee virheilmoitus "illegal hardware instruction  urh". Tämä ongelma ilmeisesti liittyy koneen arkkitehtuuriin.

Tässä on tietoja koneen arkkitehtuurista:

<img width="1181" height="626" alt="image" src="https://github.com/user-attachments/assets/9f8a3383-28b0-4767-a3f1-84579fa6624f" />

Luovutan URH:n kanssa ja kokeilen sen sijaan inspectrum -ohjelmaa.

        sudo apt install inspectrum
        inspectrum

Tiedostossa on radiosignaalin tallennus siitä, kun Nexan pistorasian kaukosäätimen valon 1 ON -nappia on painettu kolme kertaa.

Avasin tiedoston ohjelmassa ja vaihdoin Sample raten arvoksi 2000000, koska tiedoston nimessä olevasta arvosta 2MSps pystyi sen päättelemään.

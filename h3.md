# h3 NFC ja RFID

RFID eli Radio Frequency Identification ja NFC eli Near Field Communication käyttävät samaa teknologiaa, mutta eroavat toisistaan toimintasäteen perusteella. NFC on RFID:n alalaji. 
Tässä molempien käyttämässä teknologiassa on aina lukija ja tagi. Lukija luo sähkömagneettisen kentän lähettämällä radioaaltoja. Tagi koostuu dataa sisältävästä sirusta ja antennista. 
Kun tagi on lukijan luomassa sähkömagneettisessa kentässä, sen tagin antenniin indusoituu jännite, josta siru saa virran. Tällöin lukija pystyy lukemaan sirussa olevan datan. 
On myös olemassa aktiivisia RFID tageja, joilla on oma virransyöttö eivätkä ole riippuvaisia lukijan radioaalloista.

RFID:llä on paljon suurempi kantama kuin NFC:llä eli sähkömagneettinen kenttä on suurempi. Tämän vuoksi RFID:n ja NFC:n käyttötarkoitukset ovat erilaiset. RFID:tä käytetään paljon logistiikassa, 
koska se mahdollistaan nopean inventaarion tuotteissa olevien tagien ansiosta. Sillä voi myös seurata tuotannon ja toimitusketjun etenemistä. NFC:n toimintasäde on sen sijaan vain muutaman sentin. 
NFC:tä käytetään sen vuoksi maksukorttien lähimaksuominaisuudessa, passeissa ja kulkukorteissa. Tämä johtuu turvallisuussyistä, koska NFC sirujen ei haluta olla kaikkien luettavissa vaan ainoastaan 
silloin, kun se on tagin haltijan tarkoituksena.

## a) Tarkastele käytössäsi olevia RFID tuotteita, mieti miten hyvin olet suojautunut RFID urkinnalta?

Tehokkain tapa suojautua RFID urkinnalta on piilottaa tagit, joita ei halua muiden lukevan itse tietämättä siitä. Tällaisia tageja on ainakin pankkikortin lähimaksuun käytettävä tagi ja omien kulkukorttien tagit.
NFC vaatii muutaman sentin etäisyyden toimiakseen, joten usein riittää, että omat NFC tagit pitää laukun tai repun sisällä lompakossa eikä vain itsekseen taskussa. Tällöin mahdollinen urkkija ei pääse tarpeeksi lähelle tageja.

Omien tagien suojausta pystyy parantamaan myös muullakin kuin vain etäisyydellä. Tagit kannattaa laittaa metallikuoren sisälle eli esimerkiksi pankkikortit metallisen lompakon sisälle.
Tämä suojaus perustuu Faradayn häkki nimiseen ilmiöön. Kun metallinen häkki tai kotelo sijoitetaan sähkömagneettiseen kenttään, sähkömagneettinen säteily (tässä tapauksessa radioaallot) ei 
pääse häkin sisäpuolelle ([Chris Pollette & Nathan Chandler 2023: How Faraday Cages Work](https://science.howstuffworks.com/faraday-cage.htm)).

Android puhelimeen pystyy latamaan sovelluksia, joilla voi lukea RFID tageja sekä tarkastelemaan niiden sisältämää dataa. Niillä pystyy lukemaan tagin UID:n sekä saamaan selville tagin valmistajan ja mallin. Tageja pystyy kloonaamaan erityisillä laitteilla, kuten Flipper Zerolla ja Proxmark3 RDV4:lla. Tageja on myös suojaukseltaa eritasoisia. Joissain on UID suoraan luettavissa, mutta jotkut tagit käyttävät salausta.

Itselläni kulkukortit ovat aina joko lukitussa tilassa tai työpaikkalla taskussani enkä julkisissa paikoissa kanna niitä mukanani. Niiden osalta uskon olevani riittävästi suojautunut RFID urkinnalta. Maksukorttini ovat usein joko repussa tai laukussa lompakon sisällä. Voisin vaihtaa lompakkoni metalliseen, koska silloin kortit eivät olisi luettavissa, jos pidän lompakkoa taskussa.

NFC tageihin on helppo kirjoittaa itse jotain sisältöä puhelimeen ladattavan sovelluksen avulla. Niiden data osioon voi esimerkiksi lisätä jonkin nettisivun osoitteen. Kun NFC tagin vie puhelimen lähelle, puhelin lukee tagiin kirjoitetun webosoitteen ja avaa sen. Hyökkääjä voisi siis helposti sijoitella NFC tagin sellaiselle paikalle, jonka läheisyydessä puhelimia usein on ja ohjata ne jollekin haitalliselle nettisivulle (esim. nettisivulle, jossa kalastellaan tietoja). Puhelimesta voisi siis laittaa NFC lukijan pois päältä ja käyttää sitä vain tarvittaessa.

## b) Tutustu APDU komentojen rakenteeseen

APDU eli Application Protocol Data Unit on protokolla, jonka avulla älykortti ja kortinlukija kommunikoivat. Ainakin Java kortit ovat älykortteja ja niitä käytetään SIM-korteissa, passeissa ja kulkukorteissa ([CardLogix: Java Card](https://www.cardlogix.com/glossary/java-card-smart/)). Älykortteja on APDU:n rakenne on määritelty ISO/IEC 7816-4 standardissa. APDU:t muodostavat komento-vastaus parin, kun ensin toinen laite lähettää komennon, johon toinen lähettää vastauksen. Komento APDU (C-APDU) koostuu headerista eli otsikkokentästä ja valinnaisesta datakentästä (body). Vastaus APDU (R-APDU) koostuu valinnaisesta datakentästä ja trailerista eli hännästä.

C-APDU header koostuu:

- CLA: Instruction class kertoo komennon tyypin (esim. voiko eri valmistajien laitteet kommunikoida keskenään vai vain saman valmistajan)
- INS: Instruction code kertoo mikä komento on kyseessä
    - A4: Select file
    - B0: Read binary
    - D6: Update binary
    - B2: Read records
    - 20: Verify
    - 2C: Reset counter
    - 826E: Terminate card usage
    - DC: Erase records
    - 88: Internal authenticate
    - 82: External authenticate
    - 2C: Reset retry counter
    - 84: Get challenge
- P1: Instruction parameter 1 kertoo lisätietoja INS -kohdan komennolle
- P2: Instruction parameter 2 kertoo lisätietoja INS -kohdan komennolle

C-APDU body koostuu:

- Lc: Kuinka monta tavua dataa lähetetään.
- Data
- Le: Kuinka monta tavua dataa odotetaan vastauksessa.

R-APDU koostuu:

- Vastaus data
- SW1 ja SW2 komennon prosessointi statuskoodit
    - 90 + 00 &rarr; Onnistunut
    - 6A + 82 &rarr; Dataa ei löydy
    - 69 + 82 &rarr; Ei riittäviä käyttöoikeuksia

## c) Tutki ja kerro minkä mielenkiintoisen RFID hakkerointi uutiset löysit

### [Alessandro Mascellino 2026: Ghost Tap Malware Fuels Surge in Remote NFC Payment Fraud](https://www.infosecurity-magazine.com/news/ghost-tap-malware-remote-nfc-fraud/)

Uutinen kertoo uudenlaisista haittaohjelmista, jotka hyödyntävät maksukorttien NFC ominaisuutta. Tällaisia haittaohjelmia on havaittu yli 54 kappaletta. Ensin uhri huijataan asentamaan puhelimeensa haitallinen sovellus. Useat sovellukset esittävät oikeita maksusovelluksia. Sovelluksessa uhria pyydetään näyttämään maksukorttia puhelimen NFC lukijalle, jolloin maksukortin tiedot lähetetään rikolliselle. Nyt rikollinen pystyy käyttämään omalla "tapper" sovelluksellaan uhrin korttia. Rikollisen tekemät maksut näyttävät tavallisilta lähimaksuilta uhrin kortilla. Näitä "tapper" sovelluksia kaupataan palveluina Telegrammissa maksua vastaan.

Tällaiselta haittaohjelmalta voi suojautua olemalla tietoinen tekstiviesti- ja puhelinhuijauksista sekä seuraamalla omia pankkitapahtumia aktiivisesti. Pankkien tulisi myös tarkemmin suorittaa kauppiaan tarkastus sekä huomata poikkeavaisuuksia asiakkaidensa pankkitapahtumissa.
